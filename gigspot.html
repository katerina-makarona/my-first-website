GigSpot Case Study

Project Overview

This case study explores the design and development of GigSpot, a mobile app tailored for people who love music. The project aimed to address the challenge of finding live music events efficiently, considering user pain points and preferences. Through user research, competitive analysis, and iterative wireframe design, GigSpot was developed to enhance live music discovery, foster artist-audience connections, and streamline ticket booking. The focus was on creating an intuitive interface and providing personalized event recommendations, ensuring an engaging user experience for music lovers worldwide.

Problem

Many people find it difficult to discover live music events that match their interests and location. Browsing multiple websites or social media is time-consuming and often ineffective. This leads to missed events and frustration, making it hard for users to stay connected with their favourite artists and local music scenes.

Solution

GigSpot simplifies live music discovery by offering a platform tailored to user preferences like genre, date, and venue. Instead of searching multiple sites, users can find personalized concert recommendations in one place. With features like following artists and saving favorite venues, users can easily stay connected and discover events that match their tastes and location, without the hassle of browsing through irrelevant options.

Competitive Analysis

I conducted a competitive analysis of leading apps in the industry to evaluate user interfaces and identify best practices.

My Keytakeaways

Bandsintown
  • Implement a location-changing feature
  • Maintain a clean design, avoiding overcrowding the elements
  • Implement larger, clear options directly beneath the search bar
  • Provide an easy-to-access button for listening to the artist's music
  • Use a follow button as the primary action to build user connection with artists

All events
  • Make key elements like featured events or recommended categories more prominent using larger fonts to grab attention

Eventbrite
  • Implement clear categories and filtering options tailored specifically to music events for a more focused experience
  • Use easy-to-navigate search and category filters, with a simple, visually clean interface

Songkick
  • Provide a modern, visually appealing UI with an intuitive user experience
  • Go beyond event discovery by offering personalized, interactive features such as artist and venue engagement

User Personas

After conducting interviews, I developed user personas to align the design with the target audiences' needs and behaviors.

MVP Objective

To develop a core version of GigSpot that allows users to effortlessly discover live music events, follow their favourite artists, and share event details with friends. The MVP will focus on simplifying the user journey for finding events, booking tickets, and building connections with artists.

Key Features

  • Artist Profiles: Users can visit artist profiles to view upcoming events and learn more about their music
  • Follow Artists: Allow users to follow their favorite artists, receiving updates on new events and notifications
  • Event Discovery: Enable users to easily find live music events based on location, genre, and artist preferences
  • Ticket Booking: Provide a simple, user-friendly ticket booking process directly from the event page
  • Event Sharing: Allow users to quickly share event details with friends via social media or messaging platforms.
